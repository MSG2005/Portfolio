Overview

One source of truth.

A growing retail operation managed data across three POS systems, a legacy ERP, and marketing spend tracked in Google Sheets. Every Monday, an analyst spent 8 hours manually stitching together the weekly report — often from conflicting spreadsheet versions.

The goal: build an end-to-end automated analytics system that pulls all sources together, computes KPIs in real time, and delivers a boardroom-ready dashboard with zero manual intervention on report day.

"Category B products drove 61% of returns despite only 18% of revenue — a quality signal completely hidden in the noise."

Data Sources

Five inputs,
one unified model

The work began with a full source mapping exercise — every data producer documented, field definitions aligned, and a star schema designed before a single query was written.

Power Query M-code handled the heavy lifting: cleaning, merging, and standardizing all five sources into a single model with a daily scheduled refresh.

"Designing the schema first — before touching any tool — was the decision that made everything else fast."

Build Process

From scattered data to automated intelligence

Phase 01
Schema Design
Mapped all 5 sources, documented every field definition, and designed a star schema (Fact Sales + 3 dimension tables) before opening Power BI.
Data Modeling
Phase 02
Power Query ETL Pipeline
Built M-code transformations for each source — type casting, null handling, key standardization — with a daily scheduled refresh and full audit log.
Power Query · M-code
Phase 03
DAX Measures & KPI Framework
Defined 12 core KPIs — Revenue, AOV, Return Rate, Retention, Margin, CAC — using DAX with time-intelligence functions for MoM, QoQ, and YoY comparison.
DAX · Time Intelligence
Phase 04
Dashboard Design & Delivery
4 views designed for different audiences: Exec Summary, Sales Deep-Dive, Inventory Health, and Customer Cohort — each with drill-through and date slicers.
